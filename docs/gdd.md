# GDD：灰燼護航（ashes-convoy）

## 專案定位

《灰燼護航》是一款手機直式優先的像素風末日喪屍護航射擊網頁遊戲。載具可以是末日列車、飛船或後續其他載具，因此中文標題與英文 slug 不綁定單一載具型態。

- 中文名稱：灰燼護航
- 英文 slug：ashes-convoy
- 類型：護航射擊、波次生存、輕量 meta 養成
- 技術：vanilla HTML/CSS/JS + Canvas，零執行期依賴
- 主要目標裝置：手機直式 390 x 844
- 次要目標裝置：平板 820 x 1180、桌機瀏覽器
- 美術：所有圖像以程式定義 pixel matrix + 調色盤繪製到 offscreen canvas，不使用二進位圖檔或外部資源

## 一句話核心循環

選擇載具出發，拖曳瞄準自動射擊喪屍潮，擊破並吃下增益門撐過波次與 Boss，死亡後用廢土零件解鎖與升級載具，再推進到更遠的荒原。

## 設計支柱

1. 單指可玩：主要操作只需要拖曳瞄準與少量點擊。
2. 廣告感節奏：短時間看見火力變大、彈幕變密、喪屍潮爆開。
3. 真的可玩：波次、Boss、死亡結算與 meta 成長形成完整閉環。
4. 高品質像素風：角色、載具、子彈、爆炸、背景都由可測試的 sprite 合約產出。
5. 可測與可部署：核心規則純函式化，rng/now 注入，Node 單元測試與 Playwright E2E 作為 Stage Gate。

## 核心流程

1. 避難所基地選擇已解鎖載具與永久升級。
2. 進入關卡，載具自動畫面向前推進。
3. 玩家拖曳瞄準，武器依目前火力自動射擊。
4. 喪屍從前方與側翼衝來，接觸載具會造成傷害。
5. 每波中段出現增益門，玩家擊破門芯後取得臨時強化。
6. 每 5 波出現 Boss，擊敗後給較高獎勵與補給節奏。
7. 載具 HP 歸零時死亡結算，獲得廢土零件、紀錄、成就與解鎖進度。
8. 返回避難所升級，安排下一次出勤。

## 操作方式

### 手機直式

- 一指按住並拖曳：移動準星，載具武器朝準星自動射擊。
- 放開：準星停在最後方向，火力維持低頻自動射擊；再次拖曳恢復精準火力。
- 點擊增益門提示或拖曳火線射擊門芯：選定本次通過的增益門。
- 點擊 UI：暫停、技能、升級、返回車庫。
- 版面不得依賴 hover，所有互動目標最小 44 CSS px。

### 平板

- 與手機相同，一指拖曳為主。
- UI 可顯示更多資訊，例如目前火力、下一波預告、Boss 血條。
- 平板橫寬較大時仍維持直式遊戲場比例，旁側資訊不可遮擋主畫面。

### 桌機

- 滑鼠按住拖曳：瞄準與自動射擊。
- 滑鼠點擊：選增益門、按 UI。
- 鍵盤輔助：`Space` 使用主動技能、`Esc` 暫停、`1`/`2` 選擇左右增益門、`R` 重新開始。
- 桌機操作是輔助，不得成為手機無法完成的必要操作。

## 關卡與戰鬥視角

- Canvas 使用固定邏輯尺寸，建議 Stage 1 為 390 x 844 對應的邏輯座標，再由 UI 層縮放到不同裝置。
- 載具位於畫面下半部，向畫面上方推進；背景向下捲動，營造前進感。
- 喪屍從上方、左上、右上進入，少數特殊敵人會從側翼或空中切入。
- 子彈、爆炸、門與敵人都使用像素 sprite；若需要旋轉，先以 pixel matrix 預繪到 offscreen canvas，再由 Canvas 旋轉該離屏圖。

### 增益門

增益門是每波中段的短期決策。門有門芯 HP，射破門芯才生效；多個門同時出現時，只能吃到其中一個。Stage 1 先做雙門，Stage 2 擴充三門與風險門。

| id | 顯示文字 | 效果 | 疊加規則 |
|---|---|---|---|
| `damage_plus` | 火力 +35% | 本局傷害乘區 +0.35 | 加法疊加，上限總傷害 x3.5。 |
| `rate_plus` | 射速 +25% | 射擊間隔乘 0.8 | 最低間隔 0.12 秒。 |
| `multishot_plus` | 彈道 +1 | 多一條副彈道 | 上限 +4，副彈傷害 55%。 |
| `repair` | 緊急維修 | 回復最大 HP 18% | 不超過最大 HP。 |
| `shield` | 臨時護盾 | 取得最大 HP 22% 護盾 | 護盾 12 秒後衰退。 |
| `element_fire` | 燃燒彈 | 命中附加 3 秒燃燒 | 同目標最多 3 層。 |
| `element_shock` | 連鎖電 | 每 4 發觸發跳電 | 跳電傷害 40%。 |
| `element_ice` | 低溫彈 | 命中緩速 18% | Boss 緩速折半。 |

增益門不給永久貨幣，避免玩家透過重開或特定門選擇產生正收益漏洞。

## 避難所基地

Stage 3 起，meta 主畫面不再呈現平淡車庫，而是改為「溫馨避難所基地場景」（cozy shelter home-base）。玩家回到列車車廂內部的避難所，以第一人稱平視角看到床鋪、熟睡少女、補給層架、燈泡桌燈、串燈、收音機、便利貼清單、泰迪熊、植物或太陽能設備，以及車廂盡頭窗外逐漸逼近的喪屍剪影。

這個場景的目標不是展示一張靜態插圖，而是讓 meta 操作變成可理解的基地生活：

- 「出勤」或「清剿」：從車廂門口或窗邊危險區進入射擊局。
- 「升級」：對應補給層架、工具箱、便利貼清單或工作台熱區。
- 「切換載具」：對應車廂內的路線板、鑰匙牆、車輛模型或車窗旁控台。
- 「補給與成就」：對應收音機、補給層架、牆面便條或里程碑板。
- 「設定」：對應桌燈、收音機或角落小面板。

視覺情緒是室內暖橘安全與窗外冷藍危險的對比。室內用暖黃燈泡、串燈與毛毯建立庇護感；窗外用冷藍霧氣、雪或灰塵、喪屍剪影建立壓力。這是 Dark War Survival 類廣告常見的構圖與情緒參考，但本專案只用 pixel matrix sprite + Canvas 光效做分層氛圍神似，不做 AI 廣告圖的像素級複製。

少女醒著或熟睡只代表待機氛圍：

- 不影響載具數值。
- 不影響出勤收益。
- 不作為養成資源。
- 不引入照護或好感度系統。

遊戲語彙同步調整：

- 「開始戰鬥」改稱「出勤」。
- 「開始關卡」可依情境稱「前往清剿」。
- 「車庫」在玩家可見文字中改稱「避難所」或「基地」。
- 「回車庫」改稱「返回避難所」。

Stage 1 與 Stage 2 可先用簡化車庫 UI；Stage 3 必須以避難所場景取代 meta 主畫面的主視覺。

## 波次與難度曲線

### 單局節奏

- 每波長度：30 至 45 秒，依波次稍微增加。
- 一般波：敵人依預算生成。
- Boss 波：每 5 波出現 Boss，搭配少量雜兵。
- 每波至少 1 次增益門，Boss 波前一波保證出現維修或護盾選項。

### 初稿公式

```js
enemyHpScale(wave) = Math.pow(1.13, wave - 1)
enemySpeedScale(wave) = 1 + Math.min(0.45, wave * 0.015)
waveBudget(wave) = 18 + wave * 5 + Math.floor(Math.pow(wave, 1.35))
normalCountHint(wave) = 8 + Math.floor(wave * 1.6)
bossHpScale(bossIndex) = Math.pow(1.28, bossIndex - 1)
```

敵人花費初稿：

| 敵人 | 預算花費 |
|---|---:|
| 遊蕩屍 | 2 |
| 疾奔屍 | 2 |
| 腫囊屍 | 5 |
| 鐵皮屍 | 5 |
| 攀附屍 | 4 |
| 嚎叫者 | 6 |

難度乘區：

| 難度 | HP | 數量 | 獎勵 | Boss 間隔 |
|---|---:|---:|---:|---:|
| 普通 | x1.00 | x1.00 | x1.00 | 5 |
| 硬派 | x1.25 | x1.10 | x1.15 | 5 |
| 無盡 | x1.18 | x1.18 | x1.10 | 4 |

Stage 1 只開普通難度。硬派與無盡在 Stage 2 後開放，避免早期平衡面太大。

## 死亡結算與 meta 養成

### 結算輸入

- `wavesCleared`：已完整通過的波數。
- `kills`：本局擊殺數。
- `bossesDefeated`：擊敗 Boss 數量。
- `difficultyId`：難度。
- `vehicleId`：本局載具。
- `distance`：展示用距離，由波次換算。
- `now`：注入時間，寫入紀錄。

### 廢土零件獎勵

```js
baseParts = wavesCleared * 3
killParts = Math.floor(Math.min(kills, 360) / 8)
bossParts = bossesDefeated * 12
difficultyMul = { normal: 1.0, hard: 1.15, endless: 1.10 }[difficultyId]
earnedParts = Math.max(2, Math.round((baseParts + killParts + bossParts) * difficultyMul))
```

預期節奏：

| 表現 | 估計擊殺 | Boss | 普通獎勵期望 | 可購買內容 |
|---|---:|---:|---:|---|
| 新手死亡於第 3 波 | 25 | 0 | 12 | 接近第一級車體強化。 |
| 通過第 5 波 Boss | 55 | 1 | 33 | 1 局約可買第一級車體或累積武器升級。 |
| 通過第 10 波 | 130 | 2 | 70 | 2 至 3 局可買早期關鍵升級。 |
| 通過第 15 波 | 205 | 3 | 106 | 中期升級開始拉長。 |

獎勵不含任何會再次產生廢土零件的隨機箱，避免付費箱或補給箱的期望值高於成本。

### 解鎖與保底節奏

載具解鎖有兩條路：

1. 固定價格：直接用廢土零件解鎖。
2. 藍圖：Boss 掉落藍圖包，或從里程碑獲得。

Boss 藍圖初稿：

- 每個 Boss 30% 掉落 5 張未解鎖載具優先藍圖。
- 若連續 3 個 Boss 未掉落，第 4 個 Boss 保底掉落 5 張。
- 期望值約為每 Boss 1.55 張藍圖。
- `rift_hauler` 需要 20 張藍圖，普通玩家約 13 個 Boss 解鎖；若穩定通過第 10 波，約 6 至 7 局。
- `frost_wing` 需要 24 張藍圖，約 16 個 Boss 解鎖。

保底只產生藍圖，不產生廢土零件。重複藍圖轉為改裝螺栓，改裝螺栓只能用於高階載具升級，不能換回廢土零件。

### 防正收益漏洞規則

- 開局沒有入場費，因此死亡獎勵只影響進度，不會形成投注循環。
- 任何花廢土零件購買的內容，不得回吐廢土零件。
- 藍圖、改裝螺栓、成就獎勵不得轉換回主要貨幣。
- 成就與首次通關獎勵必須一次性領取，存檔需記錄已領狀態。
- 隨機獎勵的保底只保證內容品質，不保證主貨幣返還。
- 難度獎勵乘區不得高到讓低波數硬派刷法優於正常推進；Stage 2 開放新難度前需用模擬驗證每分鐘收益。

## UI 與可見文字

- 遊戲內所有可見文字使用繁體中文與台灣用語。
- HUD 基本資訊：載具 HP、護盾、波次、擊殺、本局增益、Boss 血條。
- 死亡結算：本局波次、擊殺、Boss、獲得廢土零件、最佳紀錄、升級 CTA。
- 避難所：載具選擇、升級、解鎖進度、存檔狀態與出勤入口。
- 文字不得遮擋戰場。手機上 HUD 使用邊緣小面板與圖示化按鈕。

## 測試掛鉤方向

Stage 1 實作時需提供 `window.__test`，最少包含：

```js
window.__test = {
  getState,
  setState,
  step,
  spawnEnemy,
  damageVehicle,
  finishRun,
  grantGate,
  clearStorage,
  config,
  sprites
};
```

掛鉤只在測試環境使用，但可存在於正式頁面；不得破壞正常遊戲流程。
