# 《灰燼護航》內容深度擴充路線圖（R1）

> **文件定位**：資深遊戲設計師＋敘事總監的「內容豐富度／故事性／耐玩度」設計審查與可落地擴充計畫。  
> **範圍**：只規劃、不改 `src/` 程式。數值與現況以 `src/config.js`、`src/rules.js`、`src/game.js` 為準。  
> **技術約束**：vanilla JS + Canvas PWA、佔位／既有 AI 美術、既有 matrix fallback。擴充優先走 config 資料表與既有行為鉤子，不引入新引擎。  
> **平衡原則**：新內容用「新規則／新節奏」換深度，不用數值膨脹碾壓既有平衡；拖車家具總加成上限、門／補給溢出、藍圖保底等既有護欄維持。

---

## 0. 一句話現況診斷

**系統骨架已厚、體驗節奏已成立，但「第 6 波之後」內容曲線變平：敵人池滿檔、Boss 單一、敘事停在第 20 波，重玩主要靠數值成長與 meta 零件，而非新局面與新故事鉤子。**

核心情感資產（壁爐 × 熹、「拖車＝家」、「火種不熄」）品質很高，但尚未被戰鬥波次、環境事件與局外進程充分「演出」。

## 1. 現有內容庫存（以程式為準）

### 1.1 載具／武器／子彈

| 類別 | 現況 | 備註 |
|---|---|---|
| 載具 | 4：`land_rig` 壁爐裝甲車、`sky_barge` 晨光飛船、`sea_ark` 潮汐方舟、`void_runner` 星窗穿梭艇 | 各綁一環境與一武器；後三台藍圖解鎖（各 3 張） |
| 武器 | 4：`rig_cannon`、`sky_autocannon`、`ark_cannon`、`void_lance` | 一車一槍，局內不換槍 |
| 子彈 sprite | `bullet_machine`、`bullet_rocket`、`bullet_pulse` | 視覺差主要靠 sprite + 數值 |
| 局內彈種 powerup | 模式：`standard`／`spread`／`laser`／`homing`；等級 1–5 | 掉落循環 `spread → laser → homing`；改 `runMods.weaponMode`／`weaponLevel` |
| 助手火力 | `TRAILER_GUNNER`（熹）固定機槍 | dmg 6、間隔 0.9s；可設定開關顯示 |

**解讀**：火力「深度」在載具差異＋彈種循環＋門／補給疊層，**不在武器庫擴張**。玩家主觀上「新槍感」偏少。

## 2. 現有敘事盤點

### 2.1 已建立的框架（強）

| 元素 | 內容 |
|---|---|
| 主角關係 | **壁爐**（駕駛／玩家呼號）× **熹**（拖車裡的孩子／共生者） |
| 核心母題 | 「拖車就是家」「火種／壁爐」「灰裡的光」「我不走遠」 |
| 動機 | 護送與守護熹；在廢土中找「能停下來的地方」；把走過的路變成「亮亮的東西」 |
| 敘事載體 | 拖車通訊日誌 12 則（`b01`–`b12`）；家具 flavor 文案；載具命名與壁爐機砲 |
| 情感弧線 | 恐懼 → 日常化（鋪報紙、擺架）→ 家的確認 → 更深的路 → 望向群星 → 「火還沒熄」 |

### 2.2 薄弱處（真問題）

1. **世界觀近乎空白**：災難怎麼來、母巢巨屍是什麼、四環境為何互通、藍圖從哪來——戰鬥不回答。  
2. **動機只有「往前開」**：沒有具名目的地、沒有時限、沒有對手組織／ rival 車隊。  
3. **敘事與戰鬥脫節**：日誌多在局外讀；局內幾乎只有「Boss 來襲／環境事件」系統 banner。  
4. **結局過早鎖死**：`b12` 在 bestWave 20 解鎖且標「結局」，但里程碑做到 100——**故事已完、遊戲還在刷**。  
5. **角色戲只有雙人房**：沒有第三方聲音（廣播、信件、敵性智能、舊世界殘響）。  
6. **熹的助手定位未敘事化**：她能開火，但幾乎不說話；玩家難以感到「她在陪你擋」。

**薄弱度評分（主觀、供排程用）**

| 維度 | 分數 | 說明 |
|---|---:|---|
| 角色關係 | 8/10 | 對白質感已達小品標杆 |
| 世界觀厚度 | 3/10 | 主題有、設定少 |
| 戰鬥內敘事 | 2/10 | 幾乎無 |
| 結局／重玩敘事 | 2/10 | 單線 20 波收束 |
| 與 meta 連動 | 5/10 | 解鎖條件有，但章節標籤幾乎空 |

## 4. 設計支柱（擴充時不違反）

1. **短局可懂、長線可追**：一局仍是「清線→打門→撐波→Boss」，不改成章節 RTT 大作。  
2. **家與火是母題**：新敵人／事件命名與文案都回扣「護航、拖車、熹」。  
3. **config 優先**：新 ID 進 `config.js`；行為先吃既有 `behavior`／banner／story unlock。  
4. **美術可降級**：新敵優先 **重色／filter／既有 PNG 變體**；真缺圖用 matrix fallback。  
5. **不破壞平衡護欄**：家具 cap、門 maxed overflow、零件曲線、藍圖 pity 維持；新強度用「可讀弱點」交換。  
6. **敘事可跳過**：日誌與局內字幕不得鎖死操作；可設定縮短／關閉（沿用既有 reduced 設定哲學）。

## 5. 擴充路線圖總覽

| 層級 | 目標 | 工期感（1 人、熟悉專案） | 玩家感受一句話 |
|---|---|---|---|
| **P0 快贏** | 局面變豐富、字幕有溫度 | 約 1–2 週 | 「這一波跟上一波不一樣，熹還會說話」 |
| **P1 深度** | 進程、Boss 輪換、分支鉤、信件廣播 | 約 3–5 週 | 「我在為某個目的地與某個 Boss 練車」 |
| **P2 延展** | 多結局、難度敘事、元進程 | 約 4–8 週 | 「同一條灰路，我可以走出不同的家」 |

建議交付順序：**P0-A 敵人 → P0-B 彈種／門 → P0-C 局內字幕 → P0-D 事件文案 → P1 再動進程**。避免先做大結局系統卻局內仍空。

## 7. P1 深度（進程、Boss、分支、世界觀載體）

### P1-1 Boss 輪換表（每 5 波仍高潮，但有身份）

### P1-3 分支路線（輕量，非大地图）

不做開放地圖；用 **環境鏈／事件鏈** 模擬分支：

1. **載具仍決定主環境**（既有）。  
2. 新增 `ROUTE_NODES`（每 5 波一個「路口 banner」）：例如波 6／11／16 結束時 2 選 1：

4. 局外日誌依 `routeTag` 解鎖 **分支句**（同一 beat 兩段 lines，或 `b13_route_rift` / `b13_route_ruin`）。

**預期體驗**：兩次 bestWave 20 的「路感」不同，開始有重玩敘事差。

## 8. P2 延展（多結局、元進程、長線身份）

### P2-1 多結局（3+1）

### P2-4 第二弧主線綱要（寫作備忘，供實作切段）

**弧名：《灰燼航線》**

## 14. 結語

《灰燼護航》已經擁有多數小品做不到的兩樣東西：**可玩的護航循環**，以及 **值得留下的雙人關係**。  
內容深度不足，不是因為「系統太少」，而是因為 **高潮身份單一、中後期敵池無分層、敘事在 20 波收束後把玩家丟回純數值**。

本 R1 路線圖選擇：

1. 先用 **新敵解題 + 彈種 + 字幕 + 事件具名** 把「每一波的意義」拉起來（P0）；  
2. 再用 **Boss 輪換、契約、輕分支、信件廣播** 給長線目標（P1）；  
3. 最後才用 **多結局與元圖鑑** 把重玩變成「不同的家」（P2）。

一切以既有 vanilla 架構與平衡護欄為邊界——**加的是世界與抉擇，不是無限百分比。**
