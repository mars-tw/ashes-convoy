## 0. 技術量測摘要

方法：Pillow 讀取實際檔案（尺寸、四角 alpha、不透明像素 mean RGB、chroma 綠／洋紅殘留、bbox 填充率、縮到 32px 高後剪影 edge density、靜態圖 vs walk atlas 色差）。**未改任何檔案。**

| 檢查項 | R71 結果 | 解讀 |
|--------|----------|------|
| 敵人靜態四角 alpha | 全 `0` | 透明底乾淨 |
| chroma 綠／洋紅 bleed | 全 `0.000` | 去背成功，無 key 色殘影 |
| 房間四角 alpha | 全 `255` | 滿幅不透明底，符合局外場景 |
| 靜態 vs atlas mean RGB Δ | 0.6–3.6 | 配色管線一致，atlas 非舊圖殘留 |
| 威脅色階 | 灰綠 → 鐵鏽橙 → 毒黃核心 → 暗紅骨甲＋熔金 | 與 R71 宣稱一致 |
| 32px 剪影 edge dens（R71 四敵） | 0.24–0.32 | 小尺寸仍可讀 |
| 未重繪敵 edge dens | tar_brute **0.17**、swarm／shield ~0.17–0.23 | 戰鬥中最糊的一批 |
| `base_escape_pod.png` 體積 | **1.24 MiB**／780×900 | 單檔偏肥；品質高但吃離線快取 |
| 全庫 PNG | **~8.29 MiB** | shelter 3.12 + env 1.90 + vehicles 1.50 為大宗 |

## 2. 全庫剩餘短板（相對 R71 基準）

### 2.1 角色身份分裂（最高敘事風險）

同一「熹」在三處三張臉：

| 資產 | 髮／裝 | 畫風 | 用途 |
|------|--------|------|------|
| `story/xi.png` | 短深棕、扁平披肩 | 極簡幾何 | 歷史人設參照 |
| `shelter/.../base_escape_pod.png` | 短深棕、棕披肩 | 高品質 painterly-pixel | 局外家 |

玩家最常看到的是砲座熹，與房間熹未同一人。這不是解析度問題，是 **canonical 未鎖**。

## 4. 重繪清單 P0–P2 ＋ 規格

> 規格預設：**crisp pixel art（高解析母圖 → 單次 Lanczos 下採樣）**、暖灰燼調、透明主體 PNG、無文字／浮水印、禁止第二種畫風混入同角色。  
> 母圖建議 3–4× runtime，再 bake 到下列接口尺寸以保住 guard。

### P0 — 下一輪必做（身份／首屏／戰鬥可讀）
